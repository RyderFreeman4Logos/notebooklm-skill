# 策略：日文轻小说及外文翻译 (Japanese Light Novels & Translations)

## 背景与痛点
日文轻小说（或其中文翻译版）与传统中文小说相比，有两个独特的挑战：
1.  **极高比例的对话与短句：** 轻小说中充斥着大量的换行和短对话。如果不加处理直接堆砌，很容易触发解析器的反作弊或产生过多的无意义 Chunk。
2.  **特殊的排版符号：** 如「」、【】、《》、甚至颜文字。
3.  **日文的 Token 消耗率：** 日文（尤其是平假名和片假名）的 Token 消耗量通常高于中英文。相同的物理字数，日文极易更快地撞到 NotebookLM 的 100 万 Token 上限。

## 核心策略：对话保留与极度保守阈值

1.  **Token 阈值下调 (极度保守)**
    *   使用 `tokuin` 计算 Token 时（`--model gemini-1.5-pro`），单源的阈值应当比普通中文小说更低。
    *   **建议安全阈值：600,000 - 700,000 Token**（留出更多的缓冲余地以应对高密度的对话字符转换开销）。

2.  **语义切分点**
    *   与中文小说类似，必须从“章节”或“卷”的标题处切分。
    *   日文/轻小说常用标题正则参考：
        `^(#\s+)?(第.*?[章话幕].*|.*?[①②③④⑤⑥⑦⑧⑨⑩].*|序章|终章|幕间).*$`
    *   **绝对禁止：** 在连续的对话块（连续多行以「」或“”包围的文本）中间切断。即便达到了 Token 阈值，也必须顺延寻找下一个非对话的普通叙事段落或章节标题再切。

3.  **清理特殊插图标记**
    *   如果源文本是从 EPUB 或网站扒下来的，经常会带有类似 `[插图]` 或 `<img src=...>` 的占位符。
    *   可以通过正则 `\[(插图|图片|img)\]` 等提前清洗掉这些无效占位符，以免干扰大模型的注意力。

4.  **有序命名**
    *   `01_无职转生_第1卷_第1话-第10话.md`
