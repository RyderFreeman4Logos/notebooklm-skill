# 策略：视频字幕集 (Video Subtitles - SRT/VTT)

## 背景与痛点
很多用户希望把几百集动漫、网课、纪录片的字幕（SRT / VTT）塞给 NotebookLM 做总结或查询。
字幕文件最大的问题是：**严重碎片化**。每一句话都带有一组时间戳，导致整个文本像一份密密麻麻的账单。这会极大消耗 Token 且阻碍连贯性理解。

## 核心策略：时间戳清洗与对话合并

1.  **清洗与转换 (清理 SRT/VTT 噪音)**
    *   绝不能直接把原生 `.srt` 或 `.vtt` 文件几百个拼在一起传给 NotebookLM。时间线标签（如 `00:01:23,450 --> 00:01:25,120`）占用了至少 40% 的冗余 Token。
    *   **处理方法：** 写脚本剔除无用的序号。如果是为了阅读剧情，可以只保留纯文本内容。

2.  **重组为 Markdown**
    *   按剧集或课程小节进行分组，使用 `# ` 生成标题。
    *   *示例：*
        ```markdown
        # 第01集：机器学习简介
        大家好，欢迎来到本课程...
        今天我们将讨论...
        
        # 第02集：线性回归
        上一节课我们讲了...
        ```

3.  **Token 计价与切分**
    *   清洗后的纯文本字幕极其紧凑。此时按常规文本的 **700,000 Token** 安全线（`tokuin --model gemini-1.5-pro`）进行切分即可。
    *   **切分点：** 必须在 `# ` 剧集标题处切分，绝不能在一集内部切断。

4.  **特殊情况：如果必须保留时间戳以备引用**
    *   如果用户明确要求 AI 回答时必须给出“原视频几分几秒”，那么就不能剔除时间戳。
    *   此时，把时间戳简化成极致紧凑的格式（例如 `[01:23] 你好`），然后利用 `tokuin` 严控 Token 阈值，这通常意味着每 15-20 集就要切分一次作为独立的 Markdown 源。
    *   命名规范：`01_深度学习课程_Ep1-Ep15.md`
